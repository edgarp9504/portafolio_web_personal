Sobre mí

No solo escribo código. Resuelvo problemas de negocio.

Soy Edgar Pérez, Data Engineer y Cloud Consultant basado en Cancún. Llevo más de 5 años construyendo la infraestructura de datos que permite a empresas tomar decisiones más rápidas y con mayor certeza.

Comencé en el mundo del desarrollo backend con Python y FastAPI, y naturalmente migré hacia la ingeniería de datos cuando vi que el mayor problema de mis clientes no era la falta de datos — era que sus datos no estaban conectados, limpios ni listos para usarse.

Hoy combino experiencia técnica profunda en Azure y AWS con una visión de negocio clara: cada pipeline que construyo debe generar ROI visible y medible.

AWS · ETL

Pipeline ETL con AWS Glue para E-commerce

🛒
Problema

Una tienda en línea con datos de ventas, inventario y logística en 4 sistemas distintos. Los reportes tardaban 3 días en generarse de forma manual.

Solución

Pipeline ETL en AWS Glue que extrae datos de múltiples fuentes (S3, RDS, APIs), transforma con PySpark y carga en Redshift. Orquestado con Step Functions.

⚡ Reportes de 3 días → 15 minutos automáticos
AWS GluePySpark S3RedshiftStep Functions
Azure · Lakehouse

Arquitectura Medallion en Azure Databricks

🏔️
Problema

Empresa con múltiples fuentes de datos sin gobernanza. Los analistas perdían horas buscando la "versión correcta" de los datos antes de poder trabajar.

Solución

Implementación completa de arquitectura Medallion (Bronze/Silver/Gold) sobre Azure Data Lake con Databricks, Delta Lake y Azure Data Factory como orquestador.
